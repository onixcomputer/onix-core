## Why

Two parallel color systems (`config.colors` for CLI tools, `config.theme.colors` for desktop) serve 17 downstream consumers with overlapping fields, duplicated hex conversion utilities, and no validation on color values. Only 2 of those 17 apps (niri, kitty) update at runtime when Noctalia changes colors — the other 15 are frozen at build time. Unifying them into a single NCL-defined palette and wiring all consumers through Noctalia's runtime theming gives us contract-validated color data, eliminates the duplication, and makes wallpaper/dark-mode changes propagate system-wide.

## What Changes

- **BREAKING**: Replace `config.colors` and `config.theme.colors` with a single `config.theme` option backed by NCL-exported data
- New NCL color contracts validate hex format, required fields, and theme schema completeness at `ncl export` time
- NCL pre-computes all derived color forms (hex, noHash, rgb, ansi) — eliminates runtime utility functions from Nix
- Desktop theme definitions (tokyo-night, onix-dark, onix-light, everblush, solarized-dark) rewritten as NCL files using `mk-theme.ncl` builder
- CLI sub-palettes (editor, zen, rainbow, btop, waybar, misc) folded into the theme schema as optional extensions
- Package-dependent fields (GTK packages, wallpaper fetchurl) consolidated in a `themePackages` map in `theme.nix`
- All 17 consumer modules updated to use unified `config.theme` API
- All consumer configs restructured to support Noctalia runtime override (include pattern or config file rewrite)
- New `noctalia-theme-sync` script reads Noctalia's color state and writes override files for all apps
- Noctalia hooks (`wallpaperChange`, `darkModeChange`) wired to call `noctalia-theme-sync` — all 17 apps update live

## Capabilities

### New Capabilities
- `ncl-color-contracts`: NCL contracts for hex colors, theme schema, and derived-form generation
- `unified-theme-option`: Single home-manager option replacing both `config.colors` and `config.theme.colors`
- `noctalia-theme-sync`: Runtime color propagation from Noctalia to all themed apps

## Impact

- **17 consumer modules** change from `config.colors.*` or `config.theme.colors.*` to `config.theme.*`
- **17 consumer configs** restructured for runtime override (include/source Noctalia-generated files)
- `inventory/home-profiles/brittonr/base/colors/` (8 files) — removed, absorbed into theme NCL
- `inventory/home-profiles/brittonr/base/colors.nix` — removed (option definition)
- `inventory/home-profiles/shared/desktop/themes/mk-theme.nix` — replaced by `mk-theme.ncl`
- `inventory/home-profiles/shared/desktop/themes/*.nix` (5 theme files) — rewritten as NCL
- `inventory/home-profiles/shared/desktop/theme.nix` — rewritten to consume NCL export + themePackages map
- `inventory/home-profiles/brittonr/noctalia/noctalia-sections/extras.nix` — hooks updated
- Utility functions (`noHash`, `hexToRgb`, `hexToAnsi`) move from Nix to NCL pre-computation
- New `noctalia-theme-sync` script generated by Nix, called by Noctalia hooks
- Existing `lib/wasm.nix` `evalNickelFile` infrastructure reused — no new build dependencies

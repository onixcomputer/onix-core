# OpenTofu Helper Scripts Testing

This directory contains comprehensive tests for the abstracted helper scripts generated by the OpenTofu library.

## Quick Start

Run all tests:
```bash
./run-all-tests.sh
```

Run individual test categories:
```bash
./basic-functionality.sh
./service-integration.sh
```

## Test Categories

### 1. Basic Functionality (`basic-functionality.sh`)
- Script availability and permissions
- Status script output validation
- Service name verification
- State directory handling
- Lock file interaction

### 2. Service Integration (`service-integration.sh`)
- Service status detection
- Apply script service interaction
- Logs script integration
- State flag handling
- Service dependency verification
- Systemd condition handling

## What Gets Tested

### Service Name Fixes ✓
- Scripts reference `keycloak-terraform-deploy-adeci.service` (new deployment service)

- Lock removal (with confirmation)
- Stale lock detection
- Concurrent operation handling

### Status Checking ✓
- Service status display (main + deployment)
- Lock status information

- Apply script removes deploy-complete flag

### Error Handling ✓

- Invalid operations

### Backward Compatibility ✓
- Same operational functionality
- Equivalent information display
- Same safety guarantees
- Enhanced convenience features

## Testing Environment

- **Target System**: aspen1 (or any system with keycloak service deployed)
- **Instance**: adeci
- **Services**:
  - `keycloak.service` (main service)
  - `keycloak-terraform-deploy-adeci.service` (deployment service)
- **State Directory**: `/var/lib/keycloak-adeci-terraform`

## Helper Scripts Tested

1. **keycloak-tf-unlock-adeci** - Force remove terraform state locks
2. **keycloak-tf-status-adeci** - Show lock status and service health
3. **keycloak-tf-apply-adeci** - Manual terraform deployment trigger
4. **keycloak-tf-logs-adeci** - Monitor terraform execution

## Prerequisites

- Access to target system with keycloak service
- Sudo access (for some tests)
- Systemd environment
- OpenTofu library integration deployed

## Manual Testing Scenarios

After running automated tests, perform these manual scenarios:

### Concurrent Lock Testing
```bash
# Terminal 1: Start deployment
keycloak-tf-apply-adeci

# Terminal 2: Check status while running
keycloak-tf-status-adeci

# Terminal 3: Try concurrent apply
keycloak-tf-apply-adeci  # Should wait or fail gracefully

# Terminal 2: Force unlock if needed
echo 'y' | keycloak-tf-unlock-adeci
```

### Service State Testing
```bash
# Stop main service
sudo systemctl stop keycloak

# Check status
keycloak-tf-status-adeci  # Should show service stopped

# Try apply (should wait for service)
keycloak-tf-apply-adeci &

# Start service
sudo systemctl start keycloak

# Verify deployment proceeds
keycloak-tf-logs-adeci
```

### Lock Recovery Testing
```bash
# Create stale lock
sudo bash -c 'echo "PID: 99999" > /var/lib/keycloak-adeci-terraform/.terraform.lock.info'
sudo touch /var/lib/keycloak-adeci-terraform/.terraform.lock

# Check detection
keycloak-tf-status-adeci  # Should show stale lock warning

# Remove stale lock
echo 'y' | keycloak-tf-unlock-adeci
```

## Troubleshooting

### Scripts Not Found
```bash
# Check if scripts are generated
nix build .#nixosConfigurations.aspen1.config.environment.systemPackages
ls result/bin/*keycloak*

# Rebuild and switch
sudo nixos-rebuild switch --flake .#aspen1
```

### Permission Issues
```bash
# Run with sudo for system tests
sudo ./run-all-tests.sh

# Check state directory permissions
ls -la /var/lib/keycloak-adeci-terraform/
```

### Service Issues
```bash
# Check service status
systemctl status keycloak.service
systemctl status keycloak-terraform-deploy-adeci.service

# Check service definitions
systemctl cat keycloak-terraform-deploy-adeci.service
```
